1. Class & Object

class Smartphone:
    def __init__(self, brand):
        self.brand = brand

    def start(self):
        return f"{self.brand} loaded!"

phone = Smartphone("Apple")
print(phone.start())

Explanation: Smartphone behaves as the abstract layout setup. `self` serves as a proxy marker for whichever explicit variant object (like phone) gets run inside runtime environments.

2. Inheritance

class Device:
    def power_on(self):
        return "System Online"

class Laptop(Device):
    pass

mac = Laptop()
print(mac.power_on())

Explanation: Laptop inherits from Device. Consequently, any instance generated via Laptop directly accesses methods established upstream inside the base class parent configuration layout automatically.

3. Polymorphism

class Cat:
    def speak(self): return "Meow"

class Dog:
    def speak(self): return "Woof"

for animal in [Cat(), Dog()]:
    print(animal.speak())

Explanation: Both classes share an identical method named speak(). The interface loops through instances executing specific variant behaviors dynamically without needing custom type-checking logic blocks.

4. Encapsulation

class Vault:
    def __init__(self):
        self.__secret = "Gold"

    def get_secret(self):
        return self.__secret

v = Vault()
print(v.get_secret())

Explanation: Prefixing properties with double underscores (__secret) invokes name mangling in Python. This hides variables from direct outside modifications, protecting data integrity.
